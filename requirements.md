# プロジェクト概要

自動運転を中心とした技術記事を収集・日本語翻訳し、ローカルで快適に読み返せる静的サイトを構築する。


# ゴール

- 読んだ記事をストックして、いつでも見返せるようにしたい
- 記事をカテゴリ・タグ・レーティングで整理・検索できるようにしたい
- 一覧はわかりやすく構造化したトップページにまとめたい（概要画像・説明・リンク付き）
- スマホからもアクセスしたい


# 翻訳ルール

- 内容の省略禁止。要約せず、ありのままの記事を翻訳して表示する
- 記事の構造を変えることは禁止。リストはリストで、画像の位置やサブタイトルの場所など、オリジナルから変更を加えないこと。
- 自然な日本語を意識する（直訳ではなく読みやすい表現に）
- ローカルHTML作成後、元リンクと比較して文章・視覚コンテンツが十分に反映されているか確認する


# 画像・動画のルール

- 画像は必ずローカルにダウンロードし、元の位置に表示する（外部URLのまま埋め込みはしない）
- 画像の解像度は下げないこと。
- GIF・動画も省略しない。最初のフレームを画像として保存し、「ここに動画があった」とわかる形で表示する
- YouTube等の埋め込み動画はサムネイル＋再生ボタンのプレースホルダーにする（クリックで元動画を開く）
- 取得方法の詳細は know-how.md を参照


# UI・機能要件

- 記事に対してタグを自動付与する
- 記事に対してレーティングを設定できる（星1〜5）
- タグ・レーティングで絞り込み検索できる
- index.html の記事タイトルクリックでローカル翻訳版を開き、別途「↗原文」リンクも表示する


# 記事のターゲット

- 自動運転の技術記事（大手企業ブログ中心）: Waymo, Tesla, Wayve, Zoox, AutoX, Aurora, Gatik, Applied Intuition, Baidu, Pony.ai, WeRide, Motional, Nuro, Uber など
- 自動運転に影響する周辺技術: Nvidia, Arm, 生成AI技術, シミュレーション, チップ, センサー, ECU など

# 実装の注意
- 翻訳後のローカルHTMLと元リンクを比較し、文章と視覚コンテンツが適切に不足なく反映されていることを確認すること。特に画像や動画の見落としが多い。
- 記事のリンクを間違えないように。記事自体ではなく、その親ページであったり、論文本体ではなく、そのトップページから翻訳されていることが多い。本当にこれが見たい記事かどうかを改めて確認しながら、作業を進めてください。
